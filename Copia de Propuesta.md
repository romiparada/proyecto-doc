**Resumen del proyecto:**

El proyecto propone el desarrollo de un agente inteligente que automatice la creación de historias de usuario a partir de documentos de análisis variados (licitaciones, requerimientos de clientes, audios, textos, PDFs, etc.). Actualmente, este proceso se realiza de forma manual, lo que consume tiempo y recursos, y es propenso a errores como inconsistencias, ambigüedades u omisiones. La automatización de esta tarea permitirá una mayor eficiencia y precisión, ya que el agente puede procesar grandes volúmenes de información de manera instantánea y sistemática, a diferencia de un ser humano.

**Descripción del Proyecto:**

El proyecto se centrará en comprender y presentar la literatura científica en el área (generación de especificaciones de requisitos utilizando IA) y en desarrollar un prototipo que automatice la generación de especificaciones (en particular, la generación de historias de usuario a partir de ciertas entradas). 

El agente tendrá las siguientes capacidades clave:

* Procesamiento de diversos formatos de documentos: El sistema será capaz de analizar texto (PDF, Word, odt, otros) y archivos de audio.  
* Trazabilidad: Cada historia de usuario generada estará vinculada a las porciones de documentos de entrada que la definen. Esto facilita la validación.  
* Validación automática: El sistema podrá validar las historias de usuario generadas utilizando conjunto de datos predefinidos, asegurando que cumplan con los estándares de calidad.  
* Gestión de historias de usuario: El sistema permitirá la adición y gestión de nuevas historias de usuario dentro de un proyecto.  
* El sistema buscará encontrar posibles inconsistencias o ambigüedades no resueltas en las historias o en los documentos de entrada.

**Objetivos:** 

- El objetivo principal es desarrollar un prototipo funcional de un agente inteligente que genere historias de usuario para ser utilizada en un proyecto de desarrollo de software.

Los resultados esperados son:

* Estado del arte en el tema del proyecto que sea la base y sostén del prototipo a construir.  
* Un prototipo funcional que demuestre la capacidad de generar historias de usuario a partir de múltiples fuentes.  
* La capacidad de validar automáticamente las historias de usuario generadas, buscando calidad y consistencia.  
* Un agente que pueda justificar la generación de cada historia de usuario, especificando las partes de los documentos que le da origen.  
* Reducción significativa del tiempo y esfuerzo manual requerido para la creación de historias de usuario.  
* Métricas de validación sobre la calidad de las historias de usuarios generadas.
