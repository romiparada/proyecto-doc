# 逻辑衔接与方法论深度指南

## 段落级逻辑衔接（AXES 模型）

| 组成部分 | 说明 | 示例 |
|----------|------|------|
| **A**ssertion（主张） | 清晰的主题句，陈述核心观点 | "注意力机制能够提升序列建模效果。" |
| **X**ample（例证） | 支撑主张的具体证据或数据 | "实验中，注意力机制达到95%准确率。" |
| **E**xplanation（解释） | 分析证据为何支撑主张 | "这一提升源于其捕获长程依赖的能力。" |
| **S**ignificance（意义） | 与更广泛论点或下一段的联系 | "这一发现为本文架构设计提供了依据。" |

## 过渡信号词

| 关系类型 | 中文信号词 | 英文对应 |
|----------|------------|----------|
| 递进 | 此外、进一步、更重要的是 | furthermore, moreover |
| 转折 | 然而、但是、相反 | however, nevertheless |
| 因果 | 因此、由此可见、故而 | therefore, consequently |
| 顺序 | 首先、随后、最后 | first, subsequently, finally |
| 举例 | 例如、具体而言、特别是 | for instance, specifically |

## 方法论深度检查清单

- [ ] 每个主张都有证据支撑（数据、引用或逻辑推理）
- [ ] 方法选择有充分理由（为何选此方法而非其他？）
- [ ] 明确承认研究局限性
- [ ] 清晰陈述前提假设
- [ ] 可复现性细节充分（参数、数据集、评估指标）

## 常见问题

| 问题类型 | 表现 | 修正方法 |
|----------|------|----------|
| 逻辑断层 | 段落间缺乏衔接 | 添加过渡句说明段落关系 |
| 无据主张 | 断言缺乏证据支撑 | 补充引用、数据或推理 |
| 方法论浅薄 | "本文采用X"但无理由 | 解释为何X适合本问题 |
| 隐含假设 | 前提条件未明示 | 显式陈述假设条件 |

## 分章节指南

| 章节 | 逻辑衔接重点 | 方法论深度重点 |
|------|--------------|----------------|
| 绪论 | 问题→空白→贡献的流畅衔接 | 论证研究意义 |
| 相关工作 | 按主题分组，显式对比 | 定位与前人工作的关系 |
| 方法 | 步骤间逻辑递进 | 论证每个设计选择 |
| 实验 | 设置→结果→分析的流程 | 解释评估指标选择 |
| 讨论 | 发现→启示→局限的衔接 | 承认研究边界 |

## 输出格式

```latex
% 逻辑衔接（第45行）[Severity: Major] [Priority: P1]: 段落间逻辑断层
% 问题：从问题描述直接跳转到解决方案，缺乏过渡
% 原文：数据存在噪声。本文提出一种滤波方法。
% 修改后：数据存在噪声，这对后续分析造成干扰。因此，本文提出一种滤波方法以解决该问题。
% 理由：添加因果过渡，连接问题与解决方案

% 方法论深度（第78行）[Severity: Major] [Priority: P1]: 方法选择缺乏论证
% 问题：方法选择未说明理由
% 原文：本文采用ResNet作为骨干网络。
% 修改后：本文采用ResNet作为骨干网络，其残差连接结构能有效缓解梯度消失问题，且在特征提取任务中表现优异。
% 理由：用技术原理论证架构选择
```

## 最佳实践

参考 [Elsevier](https://elsevier.blog/logical-academic-writing/)、[Proof-Reading-Service](https://www.proof-reading-service.com/blogs/academic-publishing/a-guide-to-creating-clear-and-well-structured-scholarly-arguments)：

1. **一段一主题**：每段聚焦单一核心观点
2. **主题句先行**：段首即陈述本段主张
3. **证据链完整**：每个主张都需支撑（数据、引用或逻辑）
4. **显式过渡**：使用信号词标明段落关系
5. **论证而非描述**：解释"为何"，而非仅陈述"是什么"

## 标题后导语完整性 (S1)

**规则**：除摘要、目录、参考文献、致谢、附录等特殊部分外，每一章、每一节、每一个小节以及承担展开论证功能的四级标题，均应在标题后先给出一段导语，再进入列表、图表、公式或下一级标题。

**导语的最低功能**：
- 指明本层级将讨论什么；
- 说明为何要在此处讨论；
- 交代与上文的承接关系；
- 预告本层级内部的展开顺序。

**检测启发式**（脚本自动化）：
- 扫描 `\chapter`、`\section`、`\subsection`、`\subsubsection`、`\paragraph` 等标题命令；
- 若标题后首先出现的是列表、图表、公式环境、下一级标题，或没有可见正文，则判定为缺少导语；
- 若标题后仅出现非常短的句子，且不具备“本章/本节/下面/首先”等导入信号，则标记为导语可能过弱。

**修正建议**：在标题后补一段完整导语，用于定义本层级的研究对象、写作目的与行文路线，而不是直接堆砌内容。

## 文献综述质量验证 (A1-A4)

确保相关工作章节为综合性分析而非简单罗列。

### A1: 主题聚类（禁止作者/年份罗列）

**规则**：相关工作应按研究主题组织文献，而非按作者或发表年份罗列。连续3条及以上遵循"作者（年份）提出/引入..."模式的句子视为罗列。

**检测启发式**（脚本自动化）：
- 正则：`^.*?[（(]\d{4}[)）].*?(?:提出|引入|设计|开发|采用|构建|建立)`
- 阈值：连续3条及以上匹配 → Major/P1

| 模式 | 判定 |
|------|------|
| "张三（2019）提出了X。李四（2020）引入了Y。王五（2021）设计了Z。" | 罗列模式（标记） |
| "基于注意力的方法经历了持续演进...张三（2019）和李四（2020）从不同角度探索了...然而，王五（2021）指出..." | 主题综合（通过） |

**修正**：按主题分组重组，组内进行批判性对比分析。

### A2: 主题组后附批判性分析（LLM 判断）

**规则**：每个主题组后应有综合性评述句，对比或评价所引文献——而非仅仅列举。寻找评价性语言："然而"、"尽管"、"共同的不足"、"相比之下"。

*此规则需要 LLM 判断，正则无法可靠检测。*

### A3: 相关工作末尾的研究空白推导

**规则**：相关工作的最后一段必须包含明确的研究空白描述，为本研究提供动机。

**检测启发式**（脚本自动化）：
- 扫描 `related` 章节最后10行
- 关键词：`研究空白|不足|然而.*?尚未|仍然.*?(?:挑战|困难)|有待|缺乏|尚未解决|亟待|亟需|鲜有研究|未能充分`
- 无匹配 → Major/P1

### A4: 漏斗型引用密度（LLM 判断）

**规则**：引用密度应呈宽→窄→精的漏斗型：先介绍一般领域，逐步收窄到具体子问题，最后聚焦最相关的前人工作。扁平或倒置的漏斗暗示叙事结构不佳。

*此规则需要 LLM 判断来评估叙事弧线。*

## 跨章节逻辑链闭合检查 (C3)

**规则**：绪论中的贡献声明必须在结论中得到明确回应。如绪论中声明"本文提出X"或"本文的贡献包括Y"，结论中必须有相应的回应语言（"本文证明了"、"实验表明"、"验证了"）。

**检测启发式**（脚本自动化，`--cross-section` 参数）：
- 从 `introduction` 章节提取贡献关键词
- 从 `conclusion` 章节提取回应关键词
- 若绪论有贡献声明但结论无回应语言 → Major/P1（标记为 `[Script]` 观察）

| 绪论声明 | 期望的结论回应 |
|----------|--------------|
| "本文提出了一种新颖的注意力机制。" | "实验表明，所提注意力机制达到了..." |
| "本文的主要贡献为：（1）..." | "实验验证了贡献（1）..." |

**注意**：此检查本质上是启发式的，结果以观察形式呈现，而非确定性判断。
